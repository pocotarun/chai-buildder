## Overview

**Chai Builder** is a UI-first, interactive web application that allows users to seamlessly design and customize local Indian tea configurations from scratch. Featuring a real-time responsive design panel and an advanced programmatic transparent glass mug rendering engine, the application elevates simple selection inputs into a fully responsive aesthetic playground.

The application comes packed with structural micro-interactions, floating ingredient elements, adaptive milk-foam logic, and a full-scale **Paytm-style transaction confirmation layout** complete with dynamic Hindi text-to-speech notification voiceovers.

---

## Elite Core Features

*   **Programmatic Glassmorphism UI:** A tall, sleek glass mug configuration engineered purely out of CSS layers—featuring sharp top rims, linear reflection glare (`.cup-glow`), and a seamlessly merged glass handle layout.
*   **Live Price Metrics Controller:** Fully reactive automated calculation engine updating final totals based on type, scaling multipliers, and selected premium add-ons in real-time.
*   **Dynamic Milk Foam & Tint Layers:** Programmatic color blending system that updates liquid gradients depending on choices. Simulates rich milk tea shades, amber cold-brews, raw deep-brown black teas, and updates an overlay cream foam surface level on the fly.
*   **Floating Ingredient Particle Engine:** Generates isolated micro-nodes simulating real physical add-ons (like Ginger dust or Cardamom shells) floating lazily inside the cup using looping css translation animations.
*   **Adaptive Weather Steam Controls:** Intelligently checks active selections—immediately shutting off or rendering heat vapor streams invisible if a user selects a iced/cold variant.
*   **Paytm-Style Checkout Model:** Full-viewport transaction overlay activated upon order placement—featuring modern interface blur filters, a crisp success badge matrix, and synchronized voice responses (*"Chai Builder par ₹X rupaye prapt hue"*).

## 🗂️ Project Architecture

The architecture relies entirely on native, lightweight, zero-dependency browser files for rapid deployment cycles:

| Target File | Engineering Focus | Sub-Systems Managed |
| :--- | :--- | :--- |
| `index.html` | Structural Foundation | Control panels, input groups, visual scenes, and embedded SVG icon sets. |
| `style.css` | Glassmorphic Interface Skin | Custom accent color tokens, cubic-bezier scale multipliers, vapor keyframes, and layout grids. |
| `script.js` | Core Logic Orchestrator | DOM node manipulators, array mutations, active label extractors, and speech synthesis triggers. |

## 🌐 Continuous Deployment Pipeline (Pure Git)

The platform is integrated tightly with automated deployment hooks. To deploy updates directly to production servers without configuring secondary continuous delivery dashboards, utilize standard Git push protocols to main trunks:

bash
# Staging core internal modifications
git add .

# Pin execution log parameters
git commit -m "feat: updated core interactive liquid micro-interactions"

# Ship live build directly to production
git push origin main
Note: GitHub Actions will automatically read changes inside the repository and publish the revised live site version over your public GitHub Pages subdomain node within 60-90 seconds.
